Die Cvars

cg_drawTime [0|1|2]

Legt fest, ob die Lokalzeit unter der FPS-Anzeige abgebildet wird. 1 zeigt die Zeit im 24 Stunden Format an, 2 zeigt die Zeit im 12 Stunden Format mit am/pm Zusatz.

cg_drawTimeSeconds [0|1]

Legt fest, ob die Uhr Sekunden anzeigen soll oder nicht. Dazu muss natürlich cg_drawTime auf 1 oder 2 gesetzt sein.

cg_hud

Die Bezeichnung deines aktuellen HUDs (Informationsanzeige auf dem Bildschirm) das zu Beginn geladen wird.

cg_panzerhack [0|1]

Legt dein Automatikgewehr auf Waffenbank 2 statt 3, für den Falls, dass du Soldat Stufe 4 mit schwerer Waffe bist.
Standard: 1

Befehle

loadhud

Wird verwendet um ein angepasstes HUD aus einer .hud Datei innerhalb einer .pk3 zu laden. Kann außerdem verwendet werden um ein leeres (\loadhud blank) oder das normale ET HUD zu laden (\loadhud). Wenn ein HUD geladen wird, wird zuerst das Standard-HUD als Referenz aufgerufen und danach werden die Einstellungen aus der .hud Datei darüber geschrieben. Das hat zur Folge, dass die in der .hud Datei nicht ausdrücklich deaktivierten Elemente an den jeweils üblichen Positionen zu finden sind.

edithud

Dient zum anpassen des HUDs in Echtzeit. Kann außerdem dazu verwendet werden ein HUD zu skripten, damit die Serveradministratoren nicht extra .hud Dateien in .pk3s verpacken müssen.

Format: /edithud ElementName [Wert1] [Wert2] . . . . [WertX]

Derzeitige ElementNamen and (Werte) sind wie folgt:

  • ammocount (x, y, Skalierung)
  • chargebar (x, y, Breite)
  • compass (x, y, Größe)
  • upperright (y)
  • fireteam (x, y, Breite)
  • flagcov (x, y)
  • head (x, y, Breite, Höhe)
  • healthbar (x, y, Breite)
  • healthtext (x, y, Skalierung)
  • lagometer (x, y)
  • overheat (x, y, Breite, Höhe)
  • skillbox1 (x, y, Größe)
  • skillbox2 (x, y, Größe)
  • skillbox3 (x, y, Größe)
  • skillpic1 (x, y, Größe)
  • skillpic2 (x, y, Größe)
  • skillpic3 (x, y, Größe)
  • skilltext1 (x, y, Skalierung)
  • skilltext2 (x, y, Skalierung)
  • skilltext3 (x, y, Skalierung)
  • staminabar (x, y, Breite)
  • weaponcard (x, y, Größe)
  • xptext (x, y, Skalierung)
  • cpmtext (x, y, Skalierung)
  • chattext (x, y, Skalierung)
  • votefttext (x, y)
  • livesleft (x, y)

Alle Variablen, abgesehen von skilltextX sind Teil des Originalen HUDs. skillTextX steht als Ausdruck für die Erfahrungspunktstufen, z.B 0, 1, 2, 3, 4. Die skill[box,pic,text] Elemente korrespondieren wie folgt: 1 = klassenspezifische Fertigkeit, 2 = Kamperfahrung (battle sense), 3 = leichte Waffen.

Standard Skalierung Werte, dort wo sie verwendet wurden, sind in ET normalerweise auf 25 gesetzt.

upperright steht für die Spawnzeit/Spielzeit Anzeige, die Uhr und die FPS Anzeige die normalerweise rechts auf dem Bildschirm angezeigt werden.

(x,y) beziehen sich auf eine Auflösung von 640 x 480, unabhängig von der Auflösung mit der ihr ET spielt. Deshalb steht (0,0) für oben links, (640,480) für unten recht und (320, 240) ist das Fadenkreuz.

Um ein Element vollständig auszuschalten, müsst ihr den ersten Wert auf -1 setzen. Derzeit kann votefttext NICHT deaktiviert werden.

dumphud
Gibt das aktuelle HUD in einem .hud Datei tauglichen Format in der Konsole aus.

Allgemeine Änderungen

  • Neues ETpub In-Game Menü zumkonfigurieren der cg_drawTime Variable.
  • Integriert viele der "Project: Bug Fix" Fixes.
  • Unterstützung für den 'forcecvar' Server Befehl.
  • Verwendet ETpubs Code für Spielerbewegungen, wodurch die client prediction besser auf den Server abgestimmt wird.
  • Werden ET Pros 'etpromapscripts' verwendet, wird auf Battery "You are near the Back Door" nicht angezeigt, ehe diese nicht verwendet wird. (Unterstützung für EF_FAKEBMODEL).
  • Wenn ihr euch tot stellt werden ihr keine Nachricht bekommen, dass ihr angeblich verwundet seid und auf einen Sanitäter wartet.
  • Auf Servern mit g_weapons Schaltern 2 oder 4, werdet ihr nicht vom Client normalerweise ausgegebene Fehlklicken hören wenn ihr Spritze oder Beißzange unter Wasser verwendet.
  • '+lookdown' und '+lookup' Befehle sind deaktiviert, um geskriptete Anti-Rückstoß Cheats zu unterbinden.
