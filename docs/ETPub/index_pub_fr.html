cvars
cg_drawTime [0|1|2]
Premet l'affichage de l'heure locale en dessous de l'affichage des FSP.  Les valeurs possibles sont :
  • 0 - L'heure n'est pas affichée
  • 1 - L'heure est affichée au format 24 heures
  • 2 - L'heure est affichée au format 12 heures suivi de am ou pm.
cg_drawTimeSeconds [0|1]
Permet l'affichage des secondes dans l'heure. N'est utile que lorsque c_drawTime est à une valeur > 0. les valeurs possibles sont
  • 0 - Les secondes ne sont pas affichées
  • 1 - Les secondes sont affichées
cg_hud
Nom du hud courant qui sera loader au démarrage suivant du jeu.
cg_panzerhack [0|1]

Place la SMG dans le bank 2 des armes (au lieu du 3à lorsque vous jouez en soldier ayant un niveau 4 dans le skill  Heavy Weapons

La valeur par défaut est 1

Commandes
loadhud
Utilisé pour charger un hud customiser d'un fichier .hud contenu dans un fichier pk3. Autorise aussi l'utilisation de "/loadhud blank" pour charger un hud vide, ou "/loadhud" pour charger le hud par défaut d'ET. Lors du chargement  d'un hud, celui par défaut d'ET est chargé, puis les valeurs du hud chargé surchargent toutes les valeurs modifiées. Donc si un élément du HUD n'est pas spécifiquement modifié/désactivé dans le hud chargé, il apparaîtra à la place qu'il occupe dans le hud par défaut.
edithud
Utilisé pour éditer le hud en temps réel. Peut aussi être utilisé pour créer un script pour un hud afin que les admins du serveur n'est pas de fichier .hud à packager dans un .pk3.

format: /edithud nom-de-l-élément [valeur1] [valeur2] . . . . [valeurX]

Les nom-de-l-élément et les (valeurs) supportés sont:

  • ammocount (x, y, taille du texte)
  • chargebar (x, y, largeur)
  • compass (x, y, taille)
  • upperright (y)
  • fireteam (x, y, largeur)
  • flagcov (x, y)
  • head (x, y, largeur, hauteur)
  • healthbar (x, y, largeur)
  • healthtext (x, y, taille du texte)
  • lagometer (x, y)
  • overheat (x, y, largeur, hauteur)
  • skillbox1 (x, y, taille)
  • skillbox2 (x, y, taille)
  • skillbox3 (x, y, taille)
  • skillpic1 (x, y, taille)
  • skillpic2 (x, y, taille)
  • skillpic3 (x, y, taille)
  • skilltext1 (x, y, taille du texte)
  • skilltext2 (x, y, taille du texte)
  • skilltext3 (x, y, taille du texte)
  • staminabar (x, y, largeur)
  • weaponcard (x, y, taille)
  • xptext (x, y, taille du texte)
  • cpmtext (x, y, taille du texte)
  • chattext (x, y, taille du texte)
  • votefttext (x, y)
  • livesleft (x, y)

Tous les élément sont inclus dans le hud original d'ET à l'excéption des skilltextX, qui représent une forme textuelle des niveau d'XP (i.e. 0, 1, 2, 3, 4). Les éléments skill[box,pic,text] représentent respéctivement 1 = Les attributs dans la classe sélectionnée, 2 = le battle sense, 3 = les light weapons.

Les tailles par défaut des valeurs 'taille du texte" est de 25 dans le hud par défaut d'ET.

upperright représente les affichage du compteur de spawn, temps de jeux, horloge, fps normalement affichés sur la droite de l'écran.

(x,y) sont basés sur une échelle de 640x480 quelque soit la résolution courante de l'écran. Donc (0,0) est le coin en faut à gauche, (640,480) le coin en bas à droite et (320,240) la position du viseur.


Pour ne pas afficher un élément, mettez sa valeur ) -1. Actuellement le votefttext ne peut être désactivé.

dumphud
Dump le hud courant dans un format pouvant être mit dans un fichier .hud.

Modifications générales
  • Ajout d'une entrée de menu pour etpub permettant de configurer les cvars cg_drawTime.
  • Intègre beaucoup de corrections de Chruker ("Project: Bug Fix").
  • Supporte des commandes forcecvar du serveur.
  • Utilise le code d'etpub pour le déplacement des joueurs afin d'améliorer les prédictions sur les serveurs.
  • Si les scripts de map d'ETPro sont utilisés, vous ne verrez pas le message  "You are near the Back Door"  dans battery à moins que vous l'utilisiez. (support de EF_FAKEBMODEL).
  • Lorsque vous utilisez la commande /playdead, le message indiquant que vous êtes blessé  et que vous attendez  un medic n'est pas affiché.
  • Sur les serveurs utilisant les flag 2 ou 4 du cvar g_weapons, le son "prédit" lors de l'utilisation des pinces ou des seringues ne sera pas émit si vous tentez de les utiliser sous l'eau.
  • Les commandes '+lookdown' et '+lookup' sont désactivées pour éviter les tricherie s des script réduisant le recul des armes.
